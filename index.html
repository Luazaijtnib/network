Part 1
Complete System Architecture
Overall data flow, communication topology, and offline networking design for the Ladakh deployment.

What Is This System?

Imagine a smart watchman that never sleeps. A camera connected to a mini-computer (Jetson) watches the village boundary. When a brown bear appears, the computer identifies it using AI, takes a photo, and instantly sends it to every ranger's phone — completely without internet.

System Flow Overview

📷 Step 1 — Camera Captures

Camera continuously films the perimeter. Jetson processes each frame through the AI model.

🧠 Step 2 — AI Detects Bear

YOLOv8 model runs on Jetson GPU. If confidence > 70%, a bear is confirmed. Alert triggered.

📡 Step 3 — WiFi Hotspot Active

Jetson is always broadcasting a local WiFi hotspot. No router needed. It IS the router.

📱 Step 4 — App Auto-Connects

Ranger's phone (connected to Jetson WiFi) receives WebSocket push. App wakes up instantly.

🖼 Step 5 — Image + Alert Delivered

App shows: bear photo, timestamp, GPS coords, confidence score, and a map pin.

✅ Step 6 — Ranger Confirms

Ranger taps "Confirm Bear" on the app after reviewing the image.

🔔 Step 7 — Buzzer Activated

Ranger taps "Activate Siren". Command travels back to Jetson over WiFi. GPIO triggers buzzer.

📋 Step 8 — Logged & Stored

Event is saved locally on both Jetson and app — image, time, action taken. No cloud needed.

Key Design Principle

The Jetson is both the "brain" (AI detection) and the "router" (WiFi hotspot). There is NO external router. The Jetson creates its own small WiFi network, like a mobile hotspot on your phone — but permanent and automatic.

WIRING DIAGRAM (Beginner-Friendly): Jetson GPIO Header Relay Module Pin 1 [3.3V] ──────────────────► VCC Pin 6 [GND] ──────────────────► GND Pin 12 [GPIO] ──────────────────► IN (signal) Relay Module Output 12V Horn Siren COM ──────────────────► (+) Positive NO ──────────────────► 12V Battery (+) 12V Battery (-) ──► (-) Negative EXPLANATION: When Jetson sets GPIO Pin 12 = HIGH (3.3V) → Relay coil energizes → COM connects to NO (Normally Open) → 12V current flows through siren → SIREN SOUNDS! 🔔 When GPIO Pin 12 = LOW (0V) → Relay opens → No current to siren → Siren silent SAFETY: The relay electrically isolates the Jetson (3.3V logic) from the 12V siren. Part 4
Offline Networking System
How devices talk to each other without internet. WebSocket, MQTT, and offline communication explained for beginners.

How Offline Communication Works

Think of it like a walkie-talkie system, but using WiFi instead of radio waves. The Jetson is the "base station" — it creates the WiFi network. Phones connect to it. They all talk directly to each other inside this tiny private WiFi network.

NORMAL INTERNET COMMUNICATION (What we DON'T use): Phone → Router → ISP → Internet → Cloud Server → Phone ← WiFi ← Fiber ← ...very long path... ← OUR OFFLINE COMMUNICATION (What we USE): Phone → Jetson WiFi → FastAPI Server → Phone ← 192.168.100.1 ← (5ms latency, no internet!) Distance: Phone to Jetson = 2 meters to 150 meters Latency (speed): Under 10 milliseconds Works: 100% offline, 100% local, 0% internet

WebSocket — How It Works

HTTP (normal web) is like sending letters — you ask, you wait, you get a reply. WebSocket is like a phone call — once connected, both sides can talk instantly anytime. Perfect for alerts!

HTTP (OLD WAY): Phone: "Any alerts?" → Server: "No." Phone: "Any alerts?" → Server: "No." Phone: "Any alerts?" → Server: "No." (Poll every second — wastes battery) WEBSOCKET (OUR WAY): Phone connects → Server keeps connection open ← Server: "BEAR ALERT! Image attached!" ← (instant push!) ← Server: "Bear left area." ← (instant push!) (No polling — very low battery use) URL: ws://192.168.100.1:8000/ws ↑ ↑ ↑ ↑ Protocol Jetson IP Port Path

detector.py — AI Bear Detection

# detector.py — Runs YOLOv8 bear detection on camera frames
import cv2
import numpy as np
from ultralytics import YOLO
import logging
from datetime import datetime
import time
from config import MODEL_PATH, DETECTION_CONFIDENCE, ALERT_COOLDOWN

logger = logging.getLogger(__name__)

class BearDetector:
    """Loads AI model and detects bears in camera frames."""
    
    def __init__(self):
        # Load the trained YOLOv8 model from disk
        logger.info("Loading bear detection model...")
        self.model = YOLO(str(MODEL_PATH))
        self.last_alert_time = 0        # Track last alert time
        logger.info("Model loaded successfully!")
    
    def detect(self, frame: np.ndarray) -> dict:
        """
        Analyze one camera frame.
        Returns: dict with 'bear_detected', 'confidence', 'bbox'
        """
        # Run AI model on the frame
        results = self.model(frame, verbose=False)
        
        # Loop through all detected objects
        for result in results:
            for box in result.boxes:
                confidence = float(box.conf[0])  # 0.0 to 1.0
                class_id = int(box.cls[0])
                class_name = result.names[class_id]
                
                # Check if it's a bear with enough confidence
                if class_name == "bear" and confidence >= DETECTION_CONFIDENCE:
                    # Check cooldown (don't send 30 alerts per second)
                    current_time = time.time()
                    if current_time - self.last_alert_time < ALERT_COOLDOWN:
                        return {"bear_detected": False}
                    
                    self.last_alert_time = current_time
                    
                    # Get bounding box coordinates
                    x1, y1, x2, y2 = [int(v) for v in box.xyxy[0]]
                    
                    # Draw red rectangle on frame (so we can see what AI found)
                    cv2.rectangle(frame, (x1, y1), (x2, y2), (0, 0, 255), 2)
                    cv2.putText(frame, f"BEAR {confidence:.0%}",
                               (x1, y1-10), cv2.FONT_HERSHEY_SIMPLEX,
                               0.9, (0, 0, 255), 2)
                    
                    logger.warning(f"BEAR DETECTED! Confidence: {confidence:.1%}")
                    
                    return {
                        "bear_detected": True,
                        "confidence": round(confidence, 3),
                        "bbox": [x1, y1, x2, y2],
                        "annotated_frame": frame,     # Frame with red box drawn
                        "timestamp": datetime.now().isoformat()
                    }
        
        return {"bear_detected": False}

server.py — FastAPI + WebSocket Server

# server.py — HTTP API + WebSocket server for app communication
import asyncio
import base64
import json
import logging
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from fastapi.responses import JSONResponse
import cv2
from PIL import Image
import io
from config import MAX_IMAGE_SIZE, IMAGE_QUALITY

logger = logging.getLogger(__name__)
app = FastAPI(title="Bear Alert System")

# Store all connected phones/clients
connected_clients: list[WebSocket] = []

class BearAlertServer:
    
    async def send_alert_to_all(self, frame, detection_data: dict):
        """
        Compress image and send to ALL connected phones instantly.
        This is like broadcasting a radio message to everyone.
        """
        if not connected_clients:
            logger.info("No phones connected. Alert stored locally.")
            return
        
        # Step 1: Resize image to save bandwidth
        img = Image.fromarray(cv2.cvtColor(frame, cv2.COLOR_BGR2RGB))
        img.thumbnail(MAX_IMAGE_SIZE, Image.LANCZOS)
        
        # Step 2: Compress to JPEG
        buffer = io.BytesIO()
        img.save(buffer, format="JPEG", quality=IMAGE_QUALITY)
        
        # Step 3: Convert to base64 text (so JSON can carry it)
        img_b64 = base64.b64encode(buffer.getvalue()).decode()
        
        # Step 4: Build alert message
        alert_msg = json.dumps({
            "type": "bear_alert",
            "timestamp": detection_data["timestamp"],
            "confidence": detection_data["confidence"],
            "image_b64": img_b64,
            "camera_id": "cam_01",
            "location": "North Gate"
        })
        
        # Step 5: Send to all connected phones simultaneously
        dead_clients = []
        for client in connected_clients:
            try:
                await client.send_text(alert_msg)
                logger.info(f"Alert sent to {client.client}")
            except Exception as e:
                dead_clients.append(client)  # Phone disconnected
        
        # Remove disconnected phones
        for client in dead_clients:
            connected_clients.remove(client)

bear_server = BearAlertServer()

# ─────────── WebSocket Endpoint ───────────
@app.websocket("/ws")
async def websocket_endpoint(websocket: WebSocket):
    """Phones connect here and stay connected."""
    await websocket.accept()
    connected_clients.append(websocket)
    logger.info(f"Phone connected: {websocket.client} | Total: {len(connected_clients)}")
    
    try:
        while True:
            # Wait for commands from the phone
            data = await websocket.receive_text()
            command = json.loads(data)
            
            if command.get("command") == "buzzer_on":
                duration = command.get("duration", 10)
                from gpio_controller import gpio_ctrl
                gpio_ctrl.activate_buzzer(duration)
                await websocket.send_text(json.dumps({
                    "type": "command_ack",
                    "status": "buzzer_activated",
                    "duration": duration
                }))

    import Jetson.GPIO as GPIO

class GPIOController:
    
    def __init__(self):
        self.buzzer_active = False
        self._timer = None
    
    def activate_buzzer(self, duration_seconds: int = 10):
        """Turn on buzzer for specified seconds."""
        logger.warning(f"BUZZER ON — {duration_seconds} seconds")
        self.buzzer_active = True
        
        if GPIO_AVAILABLE:
            GPIO.output(BUZZER_GPIO_PIN, GPIO.HIGH)  # Pin goes HIGH = relay ON = buzzer ON
        
        # Auto-turn off after duration (runs in background thread)
        if self._timer:
            self._timer.cancel()
        self._timer = threading.Timer(duration_seconds, self.deactivate_buzzer)
        self._timer.start()
    
    def deactivate_buzzer(self):
        """Turn off buzzer."""
        logger.info("BUZZER OFF")
        self.buzzer_active = False
        if GPIO_AVAILABLE:
            GPIO.output(BUZZER_GPIO_PIN, GPIO.LOW)  # Pin LOW = relay OFF = buzzer OFF
    
    def cleanup(self):
        """Called when system shuts down — safely turn off GPIO."""
        self.deactivate_buzzer()
        if GPIO_AVAILABLE:
            GPIO.cleanup()

// websocket_service.dart — Manages connection to Jetson
import 'dart:convert';
import 'dart:async';
import 'package:web_socket_channel/web_socket_channel.dart';

class WebSocketService {
  static const String JETSON_IP = '192.168.100.1';
  static const int PORT = 8000;
  
  WebSocketChannel? _channel;
  StreamController<Map<String, dynamic>> _alertController = 
      StreamController.broadcast();
  
  // Other code can listen to this stream for alerts
  Stream<Map<String, dynamic>> get alertStream => _alertController.stream;
  
  bool _isConnected = false;
  bool get isConnected => _isConnected;
  
  Future<void> connect() async {
    try {
      final uri = Uri.parse(
        'ws://$JETSON_IP:$PORT/ws'
      );
      
      _channel = WebSocketChannel.connect(uri);
      _isConnected = true;
      
      // Listen for messages from Jetson
      _channel!.stream.listen(
        (message) => _handleMessage(message),
        onError: (e) => _handleDisconnect(),
        onDone: () => _handleDisconnect(),
      );
      
      print('✅ Connected to Jetson!');
    } catch (e) {
      _isConnected = false;
      print('❌ Connection failed: $e');
      // Auto-retry after 3 seconds
      Future.delayed(Duration(seconds: 3), connect);
    }
  }
  
  void _handleMessage(dynamic message) {
    final data = json.decode(message as String);
    
    if (data['type'] == 'bear_alert') {
      // Push to all listeners (screens that are watching)
      _alertController.add(data);
    }
  }
  
  void _handleDisconnect() {
    _isConnected = false;
    print('Disconnected. Reconnecting in 3s...');
    Future.delayed(Duration(seconds: 3), connect);
  }
  
  // Called when ranger presses "Activate Buzzer"
  void activateBuzzer({int durationSeconds = 10}) {
    if (!_isConnected) return;
    
    final command = json.encode({
      'command': 'buzzer_on',
      'duration': durationSeconds,
    });
    
    _channel!.sink.add(command);
    print('🔔 Buzzer command sent!');
  }
  
  void deactivateBuzzer() {
    if (!_isConnected) return;
    _channel!.sink.add(json.encode({'command': 'buzzer_off'}));
  }
}

class RTSPStreamer:

            "-f", "rtsp",
            "-rtsp_transport", "tcp",       # More reliable than UDP
            "rtsp://localhost:8554/live"
        ]

            stdout=subprocess.DEVNULL,
            stderr=subprocess.DEVNULL)
        logger.info("RTSP stream started: rtsp://192.168.100.1:8554/live")
    
    def stop(self):
        if self._process:
            self._process.terminate()

Antenna Placement Rules

  • Always mount antenna as HIGH as possible (every 10m height ≈ 40% more range)
  • Ensure line-of-sight to all ranger locations — no mountains/buildings blocking
  • Point directional antennas toward the area with most rangers
  • Keep antenna away from metal surfaces (causes signal reflection/loss)
  • Use weatherproof coaxial cable (LMR-400 rated) for runs over 5m
  • Mount camera facing away from sunrise/sunset to avoid glare

# lora_bridge.py — LoRa village-to-village communication
import serial
import threading
import time

class LoRaBridge:
    def __init__(self, port="/dev/ttyTHS0", baud=115200):
        self.serial = serial.Serial(port, baud, timeout=1)
        self.node_id = 1  # Unique ID for this village node
    
    def send_alert(self, confidence: float, camera_id: str):
        """Send bear alert to all other LoRa nodes."""
        msg = f"BEAR,{confidence:.2f},{camera_id}"
        # AT command format for RYLR998
        cmd = f"AT+SEND=0,{len(msg)},{msg}\r\n"
        self.serial.write(cmd.encode())
    
    def listen(self, callback):
        """Listen for alerts from other villages."""
        while True:
            line = self.serial.readline().decode().strip()
            if line.startswith("+RCV="):
                # Parse: +RCV=node_id,length,data,rssi,snr
                parts = line.split(",")
                data = parts[2] if len(parts) > 2 else ""
                if data.startswith("BEAR"):
                    callback(data)  # Trigger local alerts too!

Mesh Networking
Extend coverage across multiple villages using mesh WiFi — routers that connect to each other automatically.

What Is Mesh Networking?

Normal WiFi: Phone → Router → Cable → Internet (single point of failure). Mesh WiFi: Phone → Any Node ↔ Any Node ↔ Any Node (multiple paths — if one fails, data goes around it). Perfect for Ladakh where terrain blocks signals.

BATMAN-adv — Best Approach for Ladakh

What is BATMAN-adv?

BATMAN stands for "Better Approach To Mobile Adhoc Networking". It's Linux software that makes multiple WiFi routers act as one seamless network — like magic. A phone connected to Node A can talk to the Jetson at Node C even if they're mountains apart, as long as there's a chain of nodes between them.

# Install BATMAN-adv on each mesh node (Jetson or router)
sudo apt install batctl -y
sudo modprobe batman-adv

# Configure mesh interface (do this on every node)
sudo ip link set wlan0 down
sudo iw wlan0 set type ibss
sudo ip link set wlan0 up

# Join mesh network (same SSID + frequency on all nodes!)
sudo iw wlan0 ibss join bear-mesh 2412  # channel 1 = 2412 MHz

# Add WiFi interface to batman
sudo batctl if add wlan0
sudo ip link set bat0 up

# Set unique IP per node
# Node 1 (main village): 
sudo ip addr add 10.0.0.1/24 dev bat0

# Node 2 (relay):
sudo ip addr add 10.0.0.2/24 dev bat0

# Node 3 (remote village):
sudo ip addr add 10.0.0.3/24 dev bat0

# Check mesh neighbors (should show other nodes)
sudo batctl neighbors

# Test connectivity between nodes
ping 10.0.0.2  # Ping node 2 from node 1

BATTERY CALCULATION: Target autonomy: 3 days without sun (Ladakh snowstorms) Energy needed for 3 days: 481 Wh/day × 3 days = 1,443 Wh LiFePO4 usable capacity: 80% (never drain below 20%) Battery needed: 1,443 / 0.80 = 1,804 Wh At 12V: 1,804 / 12 = 150 Ah battery → USE: 200Ah LiFePO4 at 12V (covers 4+ days) → Cost: ~$200–$280 USD ALTERNATIVELY: Two 100Ah batteries in parallel (same voltage, double capacity, hot-swap one if the other fails)
